# 「舞者」(Dancer) 智能选股系统 - 设计方案

## 1. 项目概述

- **名称**：舞者 (Dancer)
- **定位**：智能A股推荐系统，基于多因子加权的选股引擎
- **版本**：v1.0
- **目标**：高性能、多策略、可扩展的量化选股平台

## 2. 架构设计

```
┌────────────────────────────────────────────┐
│           API Layer (FastAPI)              │  ← REST/WS接口
├────────────────────────────────────────────┤
│         Strategy Layer (策略层)             │  ← 策略插件
├────────────────────────────────────────────┤
│          Factor Layer (因子层)            │  ← TA-Lib指标
├────────────────────────────────────────────┤
│           Data Layer (数据层)            │  ← AKShare/Ashare
├────────────────────────────────────────────┤
│        Cache Layer (缓存层)              │  ← 文件缓存+内存LRU
└────────────────────────────────────────────┘
```

## 3. 核心流程

```
请求 → 数据获取 → 因子计算 → 策略评估 → 加权评分 → 排序推荐 → API响应
```

## 4. 评分方案

### 方案1：因子策略加权评分 (v1)
- 预设因子权重（如MACD 30%, RSI 20%, 成交量 50%）
- 简单直接，快速计算
- 权重可配置人工调整

### 方案2：智能学习评分 (后续)
- 历史回测学习：自动分析各因子在不同市场环境下的表现
- 权重自适应：根据历史数据动态调整因子权重
- 场景判断：结合大盘趋势（牛市/熊市/震荡市）修正

## 5. 技术选型

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 数据源 | AKShare + Ashare | 双源自动切换 |
| 指标 | TA-Lib | 200+标准指标 |
| 缓存 | 文件缓存 + 内存LRU | 简化依赖 |
| 接口 | FastAPI | REST + WebSocket |

## 6. API 接口设计

### REST API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/v1/stocks | 获取股票列表 |
| GET | /api/v1/screen | 执行筛选 |
| GET | /api/v1/factors/{code} | 获取因子值 |
| GET | /api/v1/strategies | 列出策略 |
| GET | /health | 健康检查 |

### WebSocket

| 事件 | 描述 |
|------|------|
| /ws/screen | 实时筛选进度推送 |
